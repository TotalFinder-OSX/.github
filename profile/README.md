## Overview

The macOS Finder handles everyday file management adequately but leaves a persistent set of power user requirements unmet — holding multiple folder locations open simultaneously, viewing two folders side by side, moving files with a keyboard shortcut, and applying meaningful visual organisation through coloured labels. <a href="#">TotalFinder</a> addresses every one of these gaps by injecting additional capabilities directly into the existing Finder process, so all native macOS behaviour continues to work exactly as before while the <a href="#">new features</a> are available on top.

Browser-style tabs in TotalFinder work exactly as expected — each tab holds an independent folder location, switching between tabs changes the current directory instantly, and multiple locations stay open in a single window without desktop clutter. For workflows involving several folders repeatedly — moving downloads into project directories, comparing source and destination folders, navigating across multiple project locations simultaneously — <a href="#">tabbed browsing</a> eliminates the constant window management that Finder's single-location-per-window model requires. Tabs persist across Finder restarts, restoring the previous session's open locations automatically. Command-clicking a folder opens it in a new tab, matching browser link behaviour that users already know.

The <a href="#">dual-pane mode</a> divides the Finder window into two independent side-by-side panels, each showing a different folder. Dragging files between panels transfers them directly without opening a second window. The two-panel layout makes comparison and transfer between two directories the simplest possible interaction — drag across the divider. <a href="#">Coloured labels</a> in the sidebar display as full-colour folder icons rather than small coloured dots, making colour-based organisation immediately readable across the entire sidebar list. The <a href="#">visor mode</a> slides Finder up from the screen bottom via a configurable keyboard shortcut, providing instant file system access from any application without switching contexts — a complement to the tab and dual-pane features for users who access the file system frequently during other work. Cut and paste for files, <a href="#">folders on top</a> sorting, and system folder visibility round out TotalFinder's feature set.

## Key Features

- Browser-style <a href="#">tabs</a> for multiple folder locations in one window
- <a href="#">Dual-pane</a> side-by-side layout for two folder locations simultaneously
- File <a href="#">cut and paste</a> with Command-X and Command-V support
- Coloured <a href="#">folder labels</a> as full icons in the sidebar
- <a href="#">Visor mode</a> sliding Finder from screen edge via keyboard shortcut
- <a href="#">Folders on top</a> sorting keeping directories above files
- <a href="#">System folder</a> visibility toggle for hidden directory access
- Session-persistent <a href="#">tab pinning</a> for always-accessible locations
- All native <a href="#">Finder integration</a> preserved including iCloud and AirDrop

## Additional Information

TotalFinder's direct injection into the Finder process is what makes it feel seamless. All native integrations — Quick Look, AirDrop, iCloud Drive sync badges, Finder extensions — continue to work correctly because it is the same Finder with additional capabilities, not a replacement that must reimplement integration points from scratch.

Dual-pane mode's value for file transfer is immediately apparent. Moving files between two folders that are both visible simultaneously is far faster than the open-window, navigate, drag-from-source workflow that single-pane Finder requires for operations involving two different directory locations.

Cut and paste carries disproportionate workflow value relative to its apparent simplicity. Every file move in stock Finder requires drag-and-drop, which is impractical when source and destination cannot both be visible at once. Keyboard-initiated moves resolve this completely.
